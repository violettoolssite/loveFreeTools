部署说明

  1. 登录 Cloudflare Dashboard
  2. 进入 Workers & Pages → Create Application → Create Worker
  3. 复制下方代码粘贴到 Worker 中，点击 Deploy
  4. 创建 KV 命名空间 EMAILS_KV 并绑定到 Worker
  5. 设置环境变量 API_BASE 为数据库 API 后端地址
  6. 为 Worker 绑定自定义域名（Settings → Triggers → Custom Domains）
  7. 在域名 Email Routing 中添加 Catch-all 规则指向此 Worker
💡 小贴士： 一个 Worker 可以绑定多个域名！只需在 Custom Domains 中添加多个域名，共享同一个 KV 存储。
⚠️ 重要提示： 必须有自己的域名才能接收邮件！workers.dev 子域名只能作为 API 端点，无法配置 Email Routing 收信。
域名要求： 域名必须托管在 Cloudflare（免费）。如果域名在其他注册商，需要将 NS 记录改为 Cloudflare 提供的 NS 服务器。

}
